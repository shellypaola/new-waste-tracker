## Edge Cases & Error Handling

### Achievement Edge Cases
- **Rapid actions:** Multiple achievements unlock simultaneously
- **Offline unlock:** Achievement unlocks while offline, syncs when online
- **Duplicate criteria:** User meets criteria multiple times (only unlock once)
- **Retroactive:** Achievements apply to historical data where appropriate

### Streak Edge Cases
- **Timezone changes:** Streak calculation respects user's timezone
- **Midnight edge:** Actions near midnight counted for correct day
- **App not opened:** Background check determines if streak continues
- **Data corruption:** Streak recovery mechanism in place

### Challenge Edge Cases
- **Challenge ends while active:** Proper completion/failure handling
- **User joins late:** Partial progress still tracked
- **Community goal reached early:** All participants rewarded
- **Conflicting challenges:** User can participate in multiple simultaneously

### Social Edge Cases
- **Leaderboard ties:** Tiebreaker logic (earliest achievement time)
- **Privacy changes:** Immediate effect on visibility
- **Deleted users:** Removed from leaderboards and friend lists
- **Inappropriate content:** Reporting and moderation system
